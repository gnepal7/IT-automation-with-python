# Storing Data in the Cloud
1. Block Storage :- like a hard drive
2. Object/Blob Storage :- lets you place and retrieve objects in a storage buckets.
Blob => Binary large objects
+ Presistent storage :- that are long lived and need to keep data across reboots and upgrades.
+ Ephemeral storage :- only temporary and only need to keep local data while they are running.

- Container
- Shared file-solution system :- NFS, CFS file system
- Databases :- SQL, NOSQL
- Storage class :- based on prices

Throughput => The amount of data that you can read and write in a given amount of time.
IOPS :- Input/Output Operations Per Secone
     => Measures how many reads or writes you can do in one second, no matter how much you're accessing.
Latancy

+ Hot data :- accessed frequently and stored in hot storage, can be stored on SSD, newer for 1 year.
+ Cold data :- accessed infrequently and stored in cold storage, older than 1 year.
